What is the goal?

Ursus is a barebones Jekyll theme that comes with a predetermined file and folder structure that helps kick off projects to speed things along.

I found myself consistently building out the same structure over and over again. The problem is that nothing is really proprietary. The SASS files are the source files from getbootstrap.com. They're just set up to to automatically compile through the power of SASS and Jekyll.

Preloaded Components

Some components are preloaded into the theme and file structure. The bootstrap navigation is loaded into the components folder and referenced into the theme. Meta is already prebuilt in based on description tags on pages and the site.

Things like Google Analytics are already referenced. To enable simply add the google tag ID into the field in config file. Scripts can be placed into the header, footer, or both. There's boolean options in the config to manage that.

To add scripts and styles go to the enqueue files in data and follow the file structure there. They'll loop into the global/scripts or global/styles files. So there's not need to find and enter those files.

Navigation links are held in the in navigation file in the data folder. These are pretty rudimentary and loop into the navbar and the page-footer. Social links on the other hand are all handled through a list in the config file.
